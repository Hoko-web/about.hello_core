# LP分析ツール導入ガイド

対象サイト：pilatesroom はろこあ LP

---

## 1. なぜ分析ツールを入れるのか

LPの最終ゴールは「LINE無料体験の予約」。
分析ツールを入れることで「何人来て、どこから来て、どう動いて、どこで離脱したか」が数値と映像でわかるようになる。
数字を見る → 仮説を立てる → 改善する → また計測する、のサイクルを回すことでCVR（コンバージョン率）を上げていく。

---

## 2. 見るべき指標

### 2-1. アクセス系（集客の量と質）

| 指標 | 意味 | 改善のヒント |
|---|---|---|
| PV（ページビュー） | ページが表示された回数 | そもそもの集客量を把握する |
| UU（ユニークユーザー） | 訪問した人数（重複除外） | PVとの差が大きい＝リピーターが多い |
| 流入元（チャネル） | SNS・検索・LINE・チラシQR等、どこから来たか | チラシやSNSの効果測定に直結 |
| 検索キーワード | どんな言葉でGoogle検索して来たか | 「新城市 ピラティス」等で来ているか確認 |

### 2-2. 行動系（LPの質）

| 指標 | 意味 | 改善のヒント |
|---|---|---|
| スクロール率 | ページのどこまで見られたか | 料金セクションまで到達しているか確認 |
| 滞在時間 | ページに滞在した時間 | 極端に短い（10秒以下）なら内容が刺さっていない |
| CTAクリック率 | LINE予約ボタンのクリック数 | ヘッダーとフッターどちらのボタンが押されているか |
| 離脱ポイント | どのセクションで離脱が多いか | コンテンツの順番や内容の改善材料 |

### 2-3. コンバージョン系（成果）

| 指標 | 意味 | 改善のヒント |
|---|---|---|
| CVR（コンバージョン率） | 訪問者のうち何%がLINE予約ボタンを押したか | LP全体の最重要指標 |
| デバイス別CVR | スマホ vs PCそれぞれのCVR | ターゲット（地域の女性）はほぼスマホなのでSP最適化が最重要 |

### 改善の考え方の例

- スクロール率が50%で止まっている → ファーストビューの下のコンテンツが弱い
- LINE予約ボタンのクリック率が低い → ボタンの文言・色・配置を変えてA/Bテスト
- 流入がSNSに偏っている → SEO対策（「新城市 ピラティス」での検索順位）を強化する余地がある

## 3. 導入するツール（3つ + GTM）

### 3-1. Google Analytics 4（GA4）

**役割：** 「誰が・どこから・何人来たか」を定量データで把握する

**取れるデータ：**
- PV・UU・セッション数
- 流入元（SNS・検索・直接アクセス等）
- デバイス情報（スマホ/PC/タブレット）
- ユーザーの地域
- コンバージョン（カスタムイベント設定が必要）

**仕組み：**
- 「イベントベース」の計測方式を採用
- ページビュー（`page_view`）もイベントの一種として扱う
- デフォルトで自動取得されるイベントと、自分で設定が必要なカスタムイベント（例：LINEボタンのクリック）がある
- カスタムイベントの設定にはGTM（後述）を使うのが一般的

**アカウント構造（3階層）：**

```
アカウント（最上位）← 事業用に1つ作る（例：「HOKO」）
  └── プロパティ（中間）← 1サイト＝1プロパティ（例：「はろこあ LP」）
        └── データストリーム（最下層）← ウェブ/アプリの区分
```

- アカウントは1つ、案件ごとにプロパティを追加していく
- クライアントに渡すときはプロパティ単位でアクセス権を付与できる

**測定ID：** `G-XXXXXXXXXX` という形式。サイト専用の識別子で、スクリプトタグの中に含まれる。

**導入手順：**
1. analytics.google.com にアクセス（事業用Googleアカウントで）
2. 「プロパティ」を作成（1サイト＝1プロパティ）
3. 「データストリーム」でウェブを選択してサイトURLを登録
4. 発行された計測用スクリプトタグを `<head>` 内に貼る（※GTM経由がおすすめ）

---

### 3-2. Google Search Console

**役割：** 「サイトに来る前」のGoogle検索データを把握する（GA4は「来た後」のデータ）

**取れるデータ：**
- Google検索結果に何回表示されたか（表示回数）
- 何回クリックされたか（クリック数）
- どのキーワードで検索されたか（検索クエリ）
- 平均検索順位

※ GA4からは検索キーワードのデータが取れないため、Search Consoleで補完する関係

**導入手順：**
1. search.google.com にアクセス
2. 「プロパティを追加」→ サイトURLを入力
3. **所有権の確認**を行う

**所有権の確認方法（「このサイトは本当にあなたのものか？」の確認プロセス）：**
- **HTMLタグ方式（おすすめ）** → `<meta name="google-site-verification" content="xxxxx" />` を `<head>` に追加
- HTMLファイルアップロード方式 → 指定されたHTMLファイルをサイトのルートに設置
- DNSレコード方式 → ドメインのDNS設定にTXTレコードを追加

静的LPならHTMLタグ方式が一番簡単。

**アカウント構造：**
- GA4のような階層構造はなく、サイト単位でフラットに並ぶ
- クライアントにアクセス権を渡す場合は、サイト単位で「ユーザーを追加」

---

### 3-3. Microsoft Clarity

**役割：** 「ユーザーが実際にどう動いたか」を定性データ（映像・ビジュアル）で把握する

**取れるデータ：**
- **ヒートマップ** → ページ上のどこがクリックされているかを色で可視化（赤＝多い、青＝少ない）
- **セッション録画** → ユーザーがどうスクロールして、どこで止まって、どこをタップしたかを動画で再生
- **スクロール率** → ページのどの深さまで何%のユーザーが到達したか

GA4が「数値」で教えてくれるのに対して、Clarityは「映像」で教えてくれるイメージ。

**導入手順：**
1. clarity.microsoft.com にMicrosoftアカウントでアクセス
2. プロジェクトを作成してサイトURLを登録
3. 発行された計測用スクリプトタグを `<head>` に貼る（※GTM経由でもOK）

**アカウント構造：**
- 事業用Microsoftアカウントで1つ作り、案件ごとに「プロジェクト」を追加

---

### 3-4. Google Tag Manager（GTM）

**役割：** タグ（計測用スクリプト）を一元管理するためのツール

**なぜ使うのか：**
GA4もClarityも `<head>` にスクリプトを直接貼って導入できるが、実務ではGTMを間に挟む。
GTMのスクリプトだけHTMLに貼っておけば、GA4・Clarity・その他のツールのタグは全部GTMの管理画面から追加・変更できる。HTMLを直接編集しなくて済む。

**特に便利な場面：**
「LINEボタンのクリックを計測したい」等のカスタムイベントを追加するとき、HTMLにコードを書くのではなく、GTMの管理画面で「このクラス名のボタンがクリックされたらGA4にイベントを送る」と設定できる。

## 4. 3つのツールの関係まとめ

```
┌─────────────────────────────────────────────────────┐
│                   ユーザーの行動                      │
│                                                       │
│  Google検索 ──→ サイト訪問 ──→ ページ閲覧 ──→ LINE予約 │
│                                                       │
│  ◀── Search Console ──▶◀──── GA4 ────▶              │
│         検索前のデータ      訪問後の定量データ          │
│                                                       │
│                        ◀──── Clarity ────▶           │
│                          訪問後の定性データ            │
│                         （映像・ヒートマップ）          │
└─────────────────────────────────────────────────────┘
```

| ツール | 費用 | 主な役割 |
|---|---|---|
| GA4 | 無料 | 誰が・どこから・何人来たか（数値） |
| Search Console | 無料 | Google検索でどう見えているか（SEO） |
| Clarity | 無料 | ユーザーが実際にどう動いたか（映像） |
| GTM | 無料 | 上記のタグを一元管理 |

---

## 5. 導入の全体フロー

1. GA4・Search Console・Clarityの各アカウントを作成（事業用アカウントで）
2. GTMのアカウントを作成して、GTMのタグだけHTMLの `<head>` に貼る
3. GTMの管理画面からGA4とClarityのタグを追加
4. Search ConsoleはHTML metaタグで所有権確認（これだけは直接HTMLに書く）
5. 各ダッシュボードでデータが取れていることを確認

---

## 6. サーバー移行時の影響

計測タグはHTMLファイルに埋め込まれているため、HTMLごと移行すればそのまま動く。
ただしドメインが変わる場合は再設定が必要。

| 状況 | GA4 | Clarity | Search Console |
|---|---|---|---|
| サーバー変更・ドメイン同じ | そのまま | そのまま | そのまま |
| サーバー変更・ドメインも変わる | URL更新が必要 | URL更新が必要 | 再設定が必要 |

**ドメインが変わる場合の注意点：**
- GA4：データストリームのURLを新ドメインに更新。過去データは旧ドメインとして残り、新ドメインは別カウント
- Search Console：新ドメインでプロパティ新規作成＋所有権確認のやり直し。旧ドメインの検索データは引き継がれない
- 対策：独自ドメインを早めに取得してGitHub Pagesに紐づけておけば、サーバー移行時もドメインが変わらないので全データを引き継げる

---

## 7. アカウント管理（フリーランスとしての運用）

**基本方針：** 事業用アカウントを1つ作り、全クライアントの分析ツールをそこに集約する

- GA4 → アカウント1つ、案件ごとにプロパティ追加
- Search Console → サイト単位でプロパティ追加
- Clarity → 案件ごとにプロジェクト追加

**クライアントへの権限付与：**
- クライアントがオーナー、自分は編集者権限をもらう形が正しい
- 自分のアカウントでクライアントのドメイン・サーバーを契約しない（所有権トラブル防止）
